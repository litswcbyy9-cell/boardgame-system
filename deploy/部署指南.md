# 生产部署指南（国内云 ECS + 域名 HTTPS + DeepSeek AI）

本指南基于 `deploy/docker-compose.prod.yml`，一键起 4 个容器：
Caddy(自动 HTTPS) → Web(Nginx 托管前端) → API(Node 后端) → MySQL。

## 架构

```
公网 443 ──► Caddy(自动证书) ──► Web/Nginx ──┬─► 静态前端
                                            └─► /api 反代 ──► API:9898 ──► MySQL
```

## 前置条件

- 一台国内云 ECS（Ubuntu 22.04 为例），公网 IP
- 一个域名，已添加 A 记录解析到该 IP
- 安全组放行 **80、443** 端口（80 用于证书校验，可不放行 3306/8080）

## 一、装 Docker（含国内镜像加速）

```bash
curl -fsSL https://get.docker.com | sh
sudo apt-get install -y docker-compose-plugin

# 国内镜像加速（拉取 mysql/node/caddy 镜像更快）
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json >/dev/null <<'EOF'
{ "registry-mirrors": ["https://docker.1ms.run", "https://docker.mirrors.ustc.edu.cn"] }
EOF
sudo systemctl restart docker
```

## 二、拉代码

```bash
git clone https://github.com/litswcbyy9-cell/boardgame-system.git
cd boardgame-system
```

## 三、配置环境变量（含密钥，不进仓库）

```bash
cp deploy/.env.prod.example deploy/.env.prod
vi deploy/.env.prod
```

填入：
- `SITE_DOMAIN` —— 你的域名（如 bg.example.com）
- `MYSQL_ROOT_PASSWORD` / `MYSQL_PASSWORD` —— 改成强随机串
- `OPENAI_API_KEY` —— 你的 DeepSeek Key

## 四、一键启动

```bash
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod up -d --build
```

## Production smoke test

After deployment:

```bash
E2E_BASE_URL=https://lhywork.top npm run test:e2e
curl -s https://lhywork.top/api/health
```

The smoke test checks:

- `/` returns the customer booking shell
- `/admin` returns the admin shell
- `/api/health` contains DB, migration, and AI status sections

首次启动 MySQL 会执行 `db/init/` 全部 SQL（建库 + 种子数据），耗时几分钟。
Caddy 会自动为域名申请 Let's Encrypt 证书。

## 五、验证

```bash
docker compose -f deploy/docker-compose.prod.yml ps   # 看容器状态
curl -s https://你的域名/api/health                    # 应返回 {"ok":true,"db":true}
```

浏览器访问 `https://你的域名` 即可。顾客预约页：`https://你的域名/#/customer`

## 常用运维

```bash
# 查日志
docker compose -f deploy/docker-compose.prod.yml logs -f api

# 更新代码后重新部署
git pull && docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod up -d --build

# 停止
docker compose -f deploy/docker-compose.prod.yml down

# 备份数据库
docker exec boardgame-mysql mysqldump -uroot -p$MYSQL_ROOT_PASSWORD boardgame > backup_$(date +%F).sql
```

## 安全提醒

- `deploy/.env.prod` 含密钥，已被 .gitignore 忽略，切勿提交
- 生产环境 `ALLOW_PUBLIC_REGISTER=0`（已默认），关闭公开注册
- MySQL 仅容器内网访问，未对公网暴露 3306
- 默认管理员账号请登录后立即改密码

## 企业级运维补充

### 部署前检查

```bash
npm run build
npm run deploy:check
```

### 数据库结构检查

在能访问 MySQL 的环境里执行：

```bash
DB_HOST=127.0.0.1 DB_USER=boardgame DB_PASSWORD=你的密码 DB_NAME=boardgame npm run db:check
```

如果是已有服务器数据卷，更新代码后先执行迁移：

```bash
npm run deploy:prod:migrate
```

本地 Docker 演示环境执行：

```bash
npm run deploy:local:migrate
```

### 数据库备份

```bash
DB_HOST=127.0.0.1 DB_USER=boardgame DB_PASSWORD=你的密码 DB_NAME=boardgame npm run db:backup
```

备份文件会写入 `backups/`，文件名包含日期和当前 Git commit，便于回滚定位。

### 查看健康状态

```bash
curl -s https://你的域名/api/health
```

返回内容会包含 API、数据库、迁移记录和 AI 配置状态，不会返回任何 API Key。

### 回滚代码

```bash
git log --oneline -5
git checkout <上一个稳定commit>
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod up -d --build
```
