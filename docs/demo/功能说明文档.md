# 骰子猫桌游馆管理系统功能说明文档

## 1. 项目定位

骰子猫桌游馆管理系统是一套面向单店桌游馆的经营管理系统，覆盖顾客预约、桌位运营、会员储值、桌游目录、员工权限、优惠券、结算订单、桌游租借、战绩计分、经营报表和 AI 助手。

当前版本以“单店精细化运营”为核心，不再优先做复杂多租户 SaaS。系统保留了部分租户字段作为未来扩展基础，但日常使用按一个门店、一个数据库、一套后台账号运行。

## 2. 技术架构

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端 | Vanilla JS SPA + Vite + Tailwind + DaisyUI | 单页应用，hash 路由，顾客页和后台共用入口 |
| 后端 | Node.js + Express | 统一提供 API 和静态页面托管 |
| 数据库 | MySQL 8 | 存储会员、预约、桌位、订单、战绩、桌游、租借等数据 |
| AI | OpenAI 兼容接口 | 支持 DeepSeek、OpenAI、通义、Moonshot 等兼容服务 |
| 桌面端 | Electron | 可打包 Windows 安装包 |

启动入口：

```powershell
npm run build -w web
cd server
$env:SERVE_WEB='1'
npm run start
```

访问地址：

| 页面 | 地址 |
| --- | --- |
| 顾客预约页 | `http://localhost:8788/#/customer` |
| 后台登录页 | `http://localhost:8788/#/login` |
| 后台首页 | `http://localhost:8788` |

演示账号：

## 3. 核心业务流程

### 3.1 顾客预约

顾客可在公开预约页完成预约，不需要后台账号。

主要能力：

- 查看店内桌游目录、人数、时长、难度和分类。
- 输入姓名、手机号、人数、到店时间和离店时间。
- 系统自动匹配容量合适、时间段可用的桌位。
- 提交后生成待确认预约。
- 顾客页内置 AI 客服，可回答桌游选择和预约相关问题。

安全边界：

- 未登录顾客不能直接进入后台。
- 顾客端只暴露公开预约和公开 AI 客服接口。

### 3.2 后台登录与首页

后台登录后进入运营控制台。

首页展示：

- 今日收入、活跃桌位、预约数量、会员数量等关键指标。
- 当前经营状态概览。
- 常驻 AI 桌宠入口，可随时唤起 AI 经营助手。

### 3.3 桌位运营

桌位模块是门店现场运营的主工作台。

主要能力：

- 查看桌位平面图和状态：空闲、已预约、使用中。
- 为顾客创建预约。
- 将预约办理到店入座。
- 为现场顾客直接开台。
- 结束桌位使用并结算。

结算后系统会自动：

- 写入消费订单。
- 计算会员折扣和优惠抵扣。
- 增加会员积分。
- 根据累计消费自动更新会员等级。

### 3.4 会员管理

会员模块用于维护顾客资产和长期经营数据。

主要能力：

- 新增、搜索和查看会员。
- 支持中文姓名全文搜索和手机号搜索。
- 会员充值、消费扣款。
- 查看余额、累计充值、累计消费、积分和等级。
- 查看会员历史预约。

会员等级：

| 等级 | 含义 |
| --- | --- |
| bronze | 普通会员 |
| silver | 白银会员 |
| gold | 黄金会员 |
| platinum | 铂金会员 |
| diamond | 钻石会员 |

等级会在结算后根据累计消费自动调整。

### 3.5 桌游目录管理

桌游目录是顾客预约页、推荐、租借和战绩系统的基础数据。

主要能力：

- 新增桌游。
- 编辑名称、封面、分类、人数范围、时长、难度、出版社、年份和描述。
- 删除桌游。
- 按名称、分类、描述搜索。
- 使用 AI 生成或润色桌游描述。

建议维护字段：

- 封面图：影响顾客页观感。
- 人数范围：影响预约推荐和 AI 推荐。
- 平均时长：影响顾客预期和经营排期。
- 描述：影响搜索、AI 客服和顾客转化。

### 3.6 优惠券

优惠券模块用于做基础营销。

主要能力：

- 创建满减券、折扣券、新人券等券种。
- 配置面额、使用门槛、数量和有效期。
- 结算时可挂接优惠抵扣逻辑。

当前版本重点是后台管理和结算基础，复杂发券策略可后续扩展。

### 3.7 订单与结算

结算模块记录桌位消费形成的订单。

主要能力：

- 查看订单列表。
- 查看订单金额、优惠金额、实付金额和支付状态。
- 统计订单数、收入、优惠金额和客单价。

当前支付为店内收银闭环，微信/支付宝真实支付暂未接入。

### 3.8 桌游租借服务

租借模块用于把馆内桌游资产变成可追踪库存。

主要能力：

- 为桌游添加实体副本。
- 记录条码、存放位置、押金和状态。
- 办理借出。
- 办理归还。
- 标记遗失。
- 查看可借、借出中、逾期和遗失统计。

适用场景：

- 会员带回家租借。
- 店内高价值桌游资产管理。
- 多副本桌游库存管理。

### 3.9 战绩与 ELO 计分

系统支持记录桌游对局结果，并维护玩家排行榜。

主要能力：

- 记录一场桌游对局。
- 支持多人参与者、排名和分数。
- 根据胜负结果更新 ELO。
- 排行榜可按胜率或 ELO 排序。

这个模块适合做会员粘性活动，例如月度排行榜、积分赛、主题赛季。

### 3.10 AI 助手

系统已接入 OpenAI 兼容大模型接口，可使用 DeepSeek 等服务。

主要入口：

- 后台 AI 助手页面。
- 常驻 AI 桌宠。
- 顾客预约页 AI 客服。
- 桌游描述 AI 生成。

后台 AI 能力：

- 基于门店实时数据回答经营问题。
- 结合桌游目录做推荐。
- 辅助生成桌游介绍文案。
- 给出运营建议，例如活动设计、热门桌游分析、桌位利用率优化。

顾客 AI 能力：

- 根据人数、时长、偏好推荐桌游。
- 回答预约和到店相关问题。
- 引导顾客选择更合适的桌游和时间段。

配置项位于 `server/.env`：

```env
OPENAI_API_KEY=你的大模型密钥
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
```

注意：密钥只应保存在后端环境变量中，不应写入前端代码或浏览器可见内容。

### 3.11 员工与权限

员工权限模块用于控制后台账号。

主要能力：

- 管理员创建员工账号。
- 设置员工角色：管理员或普通员工。
- 启用或停用账号。
- 防止管理员误操作把自己降权。
- 防止系统中最后一个管理员被降权或停用。

权限边界：

| 角色 | 能力 |
| --- | --- |
| 管理员 | 员工账号管理、删除桌游、删除租借副本等高风险操作 |
| 员工 | 日常预约、会员、桌位、结算、租借操作 |

## 4. 数据库能力

当前数据库包含：

- `venues`：门店信息。
- `game_tables` / `game_table_state`：桌位与状态。
- `players`：会员。
- `staff_profiles` / `app_users` / `auth_sessions`：员工、账号、登录会话。
- `reservations`：预约。
- `play_sessions`：开台记录。
- `game_records` / `game_record_participants`：战绩与参与者。
- `player_stats`：胜场、局数、ELO。
- `games`：桌游目录。
- `coupons` / `member_coupons`：优惠券。
- `orders` / `order_items`：订单。
- `points_logs`：积分流水。
- `game_copies` / `game_loans`：桌游副本和租借。

已增强能力：

- MySQL ngram 全文索引支持中文搜索。
- 结算事务保证订单、积分、会员等级一致更新。
- 租借借出/归还使用事务避免副本状态错乱。

## 5. 当前边界与后续建议

已完成但暂不强调：

- 多租户字段和部分租户隔离基础已存在，但当前产品目标改为单店版。

暂不做或后续再做：

- 微信支付、支付宝真实支付。
- 短信通知。
- 微信小程序。
- 复杂 SaaS 订阅计费。

建议下一步优先级：

1. 补充更多真实桌游封面和描述，提升顾客页质感。
2. 为演示准备 2-3 条固定预约、租借、结算数据。
3. 给 AI 助手沉淀固定提示词模板，例如“今日经营复盘”“周末活动方案”“按人数推荐桌游”。
4. 后续如要商用，再接支付、短信和小程序。

## 6. 演示重点

演示时建议突出三句话：

1. 这是一个真正围绕桌游馆日常经营做的单店系统，不只是 CRUD 后台。
2. 顾客预约、桌位运营、会员结算、桌游资产、战绩活动和 AI 助手已经打通。
3. 外部支付和短信暂未接入，但核心经营闭环已经能跑。
