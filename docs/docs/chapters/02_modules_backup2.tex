\section{数据流图（DFD）分析}

\subsection{0 层数据流图（上下文图）}

0 层数据流图展示系统与外部实体的交互关系。系统接收来自顾客、会员和管理员的请求，与 MySQL 数据库进行交互，并返回相应的结果。

主要数据流包括：
\begin{itemize}
  \item 顾客/会员：预约查询、预约/战绩操作
  \item 管理员：维护与报表查询
  \item 系统与数据库：持久化存储与查询
\end{itemize}

\section{软件功能模块图}

系统采用自顶向下的模块划分方式。顶层是整个桌游运营系统，下一层分为六个主要功能模块：

\begin{enumerate}
  \item \textbf{M1 基础数据管理}：维护门店、桌游目录、桌位等基础信息
  \item \textbf{M2 桌位平面图}：展示桌位的实时状态（空闲/预约/占用）
  \item \textbf{M3 预约管理}：处理预约的创建、取消、查询等操作
  \item \textbf{M4 对局与计费}：管理对局的开始、计时、结算
  \item \textbf{M5 战绩与排行}：录入战绩，维护排行榜
  \item \textbf{M6 统计报表}：生成收入、利用率等统计数据
\end{enumerate}
