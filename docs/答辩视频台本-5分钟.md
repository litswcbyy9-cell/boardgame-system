# 骰子猫桌游馆系统 5 分钟答辩视频台本

> 目标：在 5 分钟内讲清楚“项目能跑、架构完整、业务复杂、AI 有实际作用、工程化可部署”。  
> 演示建议：提前打开命令行、浏览器、代码目录、数据库或 Docker 界面。录制时尽量使用全屏浏览器，鼠标动作慢一点。

## 录制前准备

1. 打开项目目录：`E:\boardgame-system`
2. 准备命令行：

```bash
npm install
npm run build -w web
npm run start -w server
```

3. 准备浏览器地址：
   - 顾客端首页：`http://127.0.0.1:8788/`
   - 后台入口：`http://127.0.0.1:8788/admin`
   - 健康检查：`http://127.0.0.1:8788/api/health`
4. 后台测试账号：使用本地库已有店长账号登录。

## 录制时重点强调

- “单店版”是当前阶段的明确取舍，不做复杂多租户，先把一个门店做稳。
- AI 不直接写数据，这是安全边界；AI 通过工具读取数据库，再给建议。
- 桌游目录、排行榜、租借、预约记录来自数据库，不是前端演示假数据。
- 后台权限不是只做页面按钮，后端接口也有角色校验。
- 项目有部署、迁移、备份、健康检查和测试脚本，能体现工程成熟度。

## 如果演示时出错的兜底说法

- 如果 AI 调用失败：“大模型接口是外部服务，系统会降级返回确定性数据和规则建议，不影响预约、会员、租借这些核心业务。”
- 如果数据库连接失败：“健康检查会明确暴露 DB 异常，部署时通过 `.env` 和 Docker Compose 管理连接参数。”
- 如果页面数据为空：“这里是演示库数据量较少，业务逻辑仍会走真实数据库查询，可以通过种子数据或后台新增来补齐。”
