# 骰子猫桌游馆系统公网答辩视频台本（完整版）

> 建议时长：7-8 分钟。
> 演示定位：以公网服务器 `https://lhywork.top/` 实机演示为主，代码架构展示为辅。
> 讲解逻辑：先证明系统在线运行，再演示顾客端闭环，接着演示后台运营闭环，然后重点展示 AI 经营大脑和数据大屏，最后补充工程架构与部署能力。

## 录制前准备

录制前在服务器执行一次更新和演示数据补齐：

```bash
cd /opt/boardgame-system
git pull origin main
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod up -d --build
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod exec api npm run demo:seed
```

准备浏览器标签页：

- 顾客首页：`https://lhywork.top/`
- 后台入口：`https://lhywork.top/admin`
- 数据大屏：`https://lhywork.top/admin#/screen`
- 健康检查：`https://lhywork.top/api/health`

准备演示账号：

- 后台店长账号：使用服务器已有店长账号。
- 顾客演示账号：`19900061001 / demo12345`

## 总体叙事

这套系统可以按四层来讲：

1. 顾客层：预约、登录、我的预约、战绩、AI 导购、排行榜、租借和桌游目录。
2. 运营层：桌位预约、开台、会员、优惠券、订单计费、桌游目录、租借、员工权限。
3. 决策层：运营总览、AI 经营大脑、数据报表、数据大屏。
4. 工程层：Docker 部署、健康检查、数据库迁移、备份、审计、限流、演示数据脚本。

## 数据大屏模块讲解逻辑

录制时可以把数据大屏按这条线讲：

1. 使用场景：店长看经营状态、门店投屏展示、答辩时集中展示复杂度。
2. 数据来源：营收趋势、桌游热度、桌位利用率、AI 经营快照都来自后端接口。
3. 图表结构：KPI 先给全局结论，折线看趋势，柱状图看热度和利用率，AI 洞察给文字解释。
4. 决策价值：判断热门游戏补货、冷门桌位调整、收入波动、逾期租借和桌位占用风险。
5. 技术实现：独立 `screen.js` 页面、ECharts 渲染、全屏布局、实时钟、resize 自适应、离开页面释放图表实例。

## 录制异常兜底话术

- AI 调用失败：大模型接口依赖外部服务，核心预约、会员、租借和后台运营功能仍然由本系统数据库支撑。
- 页面数据少：执行 `npm run demo:seed` 补齐今日预约、会员、租借和收入演示数据。
- 登录态保留：后台使用 Bearer Token 本地缓存，便于员工连续操作；后续可以扩展为 HttpOnly Cookie。
- 容器异常：通过 `/api/health` 和 `docker compose logs api` 查看服务状态和错误日志。
