# 骰子猫桌游馆系统 5 分钟公网答辩视频台本

> 演示定位：以公网服务器 `https://lhywork.top/` 实机运行为主，代码架构为辅。
> 核心叙事：这不是普通管理后台，而是一套围绕桌游馆真实经营流程设计的“顾客预约 + 门店运营 + AI 经营大脑 + 数据治理 + 可部署运维”的完整系统。

## 录制前准备

在服务器先执行一次更新和演示数据补齐：

```bash
cd /opt/boardgame-system
git pull origin main
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod up -d --build
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod exec api npm run demo:seed
```

准备浏览器标签页：

- 顾客首页：`https://lhywork.top/`
- 后台入口：`https://lhywork.top/admin`
- 健康检查：`https://lhywork.top/api/health`

## 录制时建议强调的亮点

- 公网可访问：`https://lhywork.top/`，不是只在本地跑。
- Docker 编排：Caddy + Web + API + MySQL 四容器部署。
- 数据库驱动：桌游目录、热度、排行榜、租借、预约都来自数据库。
- AI 深度结合：AI 先查确定性工具数据，再调用大模型总结，不直接写业务数据。
- 真实经营闭环：预约、开台、会员、租借、权限、经营总览都能串起来。
- 工程成熟度：健康检查、迁移、备份、参数校验、限流、审计日志、演示数据脚本。

## 录制中可用的服务器命令

```bash
cd /opt/boardgame-system

# 查看容器状态
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod ps

# 查看 API 日志
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod logs --tail=80 api

# 补齐演示数据
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env.prod exec api npm run demo:seed

# 健康检查
curl -k https://lhywork.top/api/health
```

## 如果演示时出现异常

- AI 调用失败：说明“外部大模型服务不可控，但系统会降级到数据库确定性结果，不影响核心业务”。
- 页面数据少：执行 `npm run demo:seed` 补齐今日预约、会员、租借和收入演示数据。
- 登录态保留：说明“后台使用 Bearer Token 本地缓存，方便员工连续操作；生产上也可以升级 HttpOnly Cookie”。
- 某个容器异常：展示 `/api/health` 和 `docker compose logs api`，说明系统具备运维排查入口。
